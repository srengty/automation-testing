UI Automation

Selenium WebDriver Integration

Selenium WebDriver provides browser automation through a common interface:

  • Cross-browser support: Chrome, Firefox, Safari, Edge
  • Language bindings: Java, Python, C#, JavaScript
  • Element location: By ID, XPath, CSS selectors
  • Actions: Click, type, navigation, JavaScript execution
  • Waits: Implicit, explicit, and fluent wait strategies
WebDriver driver = new ChromeDriver();
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
WebElement loginButton = driver.findElement(By.id("login-btn"));
loginButton.click();

Design Pattern

Page Object Model (POM)

Page Object Model encapsulates page elements and actions into reusable classes:

  • Encapsulation: Hide implementation details
  • Maintainability: Centralized element management
  • Reusability: Common actions across tests
  • Readability: Business-focused test methods
public class LoginPage {
    private WebDriver driver;
    
    @FindBy(id = "username") 
    private WebElement usernameField;
    
    @FindBy(id = "password") 
    private WebElement passwordField;
    
    public void login(String username, String password) {
        usernameField.sendKeys(username);
        passwordField.sendKeys(password);
        loginButton.click();
    }
}
Encapsulation Maintainable DRY

Advanced Pattern

Screenplay Pattern

Screenplay pattern focuses on actors, tasks, and interactions for better test clarity:

  • Actor-based: Tests written from user perspective
  • Task composition: Complex workflows from simple tasks
  • SOLID principles: Better object-oriented design
  • Living documentation: Self-explaining test code
Actor user = Actor.named("John");

user.attemptsTo(
    Navigate.to("/login"),
    Enter.theValue("john@example.com").into(LoginForm.USERNAME),
    Enter.theValue("password123").into(LoginForm.PASSWORD),
    Click.on(LoginForm.SUBMIT_BUTTON)
);

user.should(See.that(Dashboard.WELCOME_MESSAGE, isVisible()));
Actor-based Readable Scalable

API Testing

REST-assured Framework

REST-assured provides a fluent API for testing REST services:

  • BDD-style syntax: Given-When-Then structure
  • JSON/XML support: Built-in parsing and validation
  • Authentication: OAuth, Basic Auth, API keys
  • Schema validation: JSON Schema and XML DTD
  • Request/Response logging: Detailed debugging info
given()
    .contentType(ContentType.JSON)
    .body(newUser)
.when()
    .post("/api/users")
.then()
    .statusCode(201)
    .body("name", equalTo("John Doe"))
    .body("email", notNullValue());
Fluent API Validation BDD

Mocking

Mockito for Test Isolation

Mockito enables unit testing with mock objects and behavior verification:

  • Mock creation: Interface and class mocking
  • Stubbing: Define mock behavior and return values
  • Verification: Ensure methods called with correct parameters
  • Argument matchers: Flexible parameter matching
  • Spy objects: Partial mocking of real objects
@Mock
private UserRepository userRepository;

@Test
void shouldCreateUser() {
    User user = new User("John", "john@test.com");
    when(userRepository.save(any(User.class))).thenReturn(user);
    
    User result = userService.createUser("John", "john@test.com");
    
    verify(userRepository).save(argThat(u -> u.getName().equals("John")));
    assertThat(result.getEmail()).isEqualTo("john@test.com");
}
Isolation Verification Control

CI/CD Integration

GitHub Actions

GitHub Actions provides integrated CI/CD workflows for test automation:

  • Workflow triggers: Push, PR, schedule, manual
  • Matrix builds: Multiple Java versions, browsers
  • Artifact storage: Test reports, logs, screenshots
  • Environment secrets: Secure credential management
  • Marketplace actions: Reusable workflow components
name: Test Automation
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          java-version: '17'
      - run: ./gradlew test
      - uses: actions/upload-artifact@v4
        with:
          name: test-reports
          path: build/reports/

CI/CD Alternative

Jenkins Pipeline

Jenkins provides enterprise-grade CI/CD with extensive plugin ecosystem:

  • Pipeline as Code: Jenkinsfile version control
  • Parallel execution: Distributed test runs
  • Plugin ecosystem: 1800+ plugins available
  • Blue Ocean: Modern pipeline visualization
  • Docker integration: Containerized build agents
pipeline {
    agent any
    stages {
        stage('Test') {
            parallel {
                stage('Unit Tests') {
                    steps { sh './gradlew test' }
                }
                stage('Integration Tests') {
                    steps { sh './gradlew integrationTest' }
                }
            }
        }
    }
    post {
        always {
            publishTestResults testResultsPattern: 'build/test-results/**/*.xml'
        }
    }
}

Reporting

Allure Test Reporting

Allure generates comprehensive, interactive test reports:

  • Rich visualizations: Charts, graphs, and timelines
  • Test categorization: Suites, features, and stories
  • Attachments: Screenshots, logs, and videos
  • History tracking: Trend analysis and flaky tests
  • Behavior-driven: BDD step reporting
@Test
@Description("Verify user login functionality")
@Severity(SeverityLevel.CRITICAL)
@Story("Authentication")
public void testUserLogin() {
    Allure.step("Navigate to login page", () -> {
        driver.get("/login");
    });
    
    Allure.step("Enter credentials", () -> {
        loginPage.enterCredentials("user", "pass");
    });
    
    Allure.addAttachment("Screenshot", takeScreenshot());
}
Interactive Visual Detailed

ExtentHtmlReporter htmlReporter = new ExtentHtmlReporter("extent.html");

test.log(Status.INFO, "Starting test execution");
